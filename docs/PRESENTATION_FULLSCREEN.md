# 演示文稿生成插件 - 全屏播放功能

## 功能概述

演示文稿生成插件现已支持 **Reveal.js 全屏播放**,提供专业的演示体验。

### 新增功能

1. **Reveal.js 渲染引擎**
   - 完整支持 Reveal.js Markdown 语法
   - 水平和垂直幻灯片导航 (`---` 和 `--`)
   - 幻灯片进度条和页码显示
   - 键盘快捷键支持

2. **全屏播放模式**
   - 一键进入全屏演示
   - 沉浸式播放体验
   - 自动适配屏幕尺寸

3. **交互式控制**
   - 工具栏快速操作
   - 键盘导航
   - 幻灯片概览模式

---

## 使用方法

### 1. 生成演示文稿

在学生会话中,AI 会自动根据知识点生成演示文稿:

```
你好,请根据当前知识点生成一份详细的演示文稿。
```

### 2. 查看演示文稿

生成完成后,点击 **"📊 立即查看演示文稿"** 按钮。

### 3. 全屏播放

在演示文稿查看器中,点击工具栏的 **"⛶ 全屏"** 按钮,或按键盘 **F** 键。

### 4. 导航控制

**键盘快捷键:**
- `→` / `Space` - 下一张幻灯片
- `←` - 上一张幻灯片
- `↓` - 垂直下一张
- `↑` - 垂直上一张
- `ESC` / `F` - 退出全屏
- `O` - 幻灯片概览模式
- `?` - 显示帮助

**鼠标控制:**
- 点击屏幕左右边缘切换幻灯片
- 使用底部控制按钮导航

## Markdown 语法

### 基础幻灯片分隔

```markdown
# 第一张幻灯片

这是内容

---

# 第二张幻灯片

更多内容
```

### 垂直幻灯片

```markdown
# 主题

---

## 子主题 1

--

## 子主题 2

--

## 子主题 3

---

# 下一个主题
```

### 演讲者备注

```markdown
# 幻灯片标题

内容...

> 备注: 这是演讲者备注,不会在幻灯片上显示
```

### 片段动画

```markdown
# 逐步显示

- 第一点 <!-- .element: class="fragment" -->
- 第二点 <!-- .element: class="fragment" -->
- 第三点 <!-- .element: class="fragment" -->
```

### 背景颜色

```markdown
<!-- .slide: data-background="#ff0000" -->

# 红色背景幻灯片
```

|------|------|--------|

## 全屏模式特性

### 自动适配
- 自动调整幻灯片尺寸适配屏幕
- 响应式布局,支持不同分辨率

### 退出方式
1. 按 `ESC` 键
2. 按 `F` 键
3. 浏览器原生全屏退出 (通常是 `F11`)

### 提示信息
进入全屏后,底部会显示 3 秒提示:
```
按 ESC 或 F 退出全屏
```

## 技术细节

### Reveal.js 配置

```typescript
{
  controls: true,           // 显示控制按钮
  progress: true,           // 显示进度条
  slideNumber: 'c/t',       // 显示页码 (当前/总数)
  transition: 'slide',      // 切换动画
  center: true,             // 内容居中
  keyboard: true,           // 键盘导航
  overview: true,           // 支持概览模式
  touch: true,              // 触摸支持
  autoAnimate: true,        // 自动动画
}
```

### 幻灯片分隔符

| 分隔符 | 用途 |
|--------|------|
| `---` | 水平幻灯片分隔 |
| `--` | 垂直幻灯片分隔 |
| `> 备注:` | 演讲者备注 |

## 故障排查

### 问题 1: 全屏按钮无响应

**原因:** 浏览器可能阻止了全屏请求

**解决:**
1. 确保用户有交互操作 (点击按钮)
2. 检查浏览器权限设置
3. 尝试使用 F 键快捷键

### 问题 2: 幻灯片显示不正确

**原因:** Markdown 格式问题

**解决:**
1. 检查分隔符 `---` 前后是否有空行
2. 确保使用正确的 Markdown 语法
3. 重新生成演示文稿

### 问题 3: 键盘导航不工作

**原因:** 焦点不在演示文稿区域

**解决:**
1. 点击幻灯片区域获取焦点
2. 确保没有其他输入框处于激活状态

## 最佳实践

### 1. 幻灯片设计

- 每张幻灯片只包含一个核心概念
- 使用简洁的标题和要点
- 避免过多文字,多用图表和示例

### 2. 导航结构

- 使用水平幻灯片表示主题
- 使用垂直幻灯片展开细节
- 保持层级清晰

### 3. 演讲者备注

- 为每张幻灯片添加备注
- 记录关键讲解点
- 提醒互动环节

### 4. 动画使用

- 适度使用片段动画
- 避免过度动画影响阅读
- 关键信息逐步展示
