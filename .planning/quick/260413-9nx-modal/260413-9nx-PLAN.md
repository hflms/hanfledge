# Quick Task 260413-9nx Plan

## Goal

Make model edits in
`frontend/src/app/teacher/settings/page.tsx` persist to `/system/config`
immediately when the modal **确认** button is clicked, so refreshed pages keep
the latest chat and embedding model configuration instead of losing modal-only
changes.

## Context

- The settings page already loads and saves system config through
  `apiFetch()` and `PUT /system/config`.
- `handleSaveModelModal` currently updates only local React state, while actual
  persistence is delayed until page-level `handleSave`.
- The fix must follow existing Next.js + TypeScript patterns and keep using
  `frontend/src/lib/api.ts` for API calls.

  compute the next `chatModels` / `embeddingModels` state, then persist that
  exact next state immediately before closing the modal. Preserve current toast
