# 系统管理员 (SYS_ADMIN) 操作手册

## 1. 角色定位与权限概述
系统管理员是 Hanfledge AI-Native 教育平台的最高权限角色，主要负责平台的全局运维、多租户（学校）架构搭建以及全局系统资源的监控。在权限模型中，系统管理员拥有“上帝视角”，能够访问系统中的所有资源。

## 2. 核心功能与操作指南

### 2.1 学校管理 (School Management)
*平台支持多学校入驻，由系统管理员统筹创建。*

* **功能路径**: `/api/v1/admin/schools`
* **操作步骤**:
  1. **新建学校**: 填写学校名称、唯一标识及基础配置信息。
  2. **分配初始超管**: 每个新建立的学校需要指定一位初始的“学校管理员”（`SCHOOL_ADMIN`）。
  3. **列表检索**: 查看全平台所有学校的入驻状态、活跃人数等汇总信息。

### 2.2 跨校班级与用户管理 (Global Class & User Management)
*虽然通常由学校管理员管理，但系统管理员拥有跨越学校边界的操作能力。*

* **功能路径**: `/api/v1/admin/classes`, `/api/v1/admin/users`
* **操作步骤**:
  1. **跨校新建班级**: 可以在任意指定的学校下创建新的班级。
  2. **人员调度**: 支持在全平台范围内进行单个或批量的用户创建操作（支持 `POST /users/batch`）。
  3. **权限下放**: 可以将特定的教师或用户提升为某学校的 `SCHOOL_ADMIN`。

### 2.3 平台宏观监控 (Platform Analytics)
*系统管理员可以调用所有下级（如教师、校管）的分析接口，查看全平台的知识流转和数据。*

* **支持的操作**:
  1. 查看任意学校的“知识雷达图”和“技能有效度分析”。
  2. 管理并审核全局的“插件市场”（Marketplace），处理教师提交上来的教学插件。

## 3. 注意事项
* **谨慎操作**: 系统管理员创建的全局角色和机构配置不可逆转时可能影响平台数据隔离。
* **数据安全**: 执行批量导入时，请确保 `json` 格式符合系统标准规范，不要包含明文敏感密码。
