# 学校管理员 (SCHOOL_ADMIN) 操作手册

## 1. 角色定位与权限概述
学校管理员是入驻本平台的单体学校（Tenant）内部的最高权限拥有者。负责组织和管理本校内部的教学活动、教师团队以及学生账号。

## 2. 核心功能与操作指南

### 2.1 师生用户管理 (User Management)
*平台采用角色隔离体系，本校的学生和教师账号应由学校管理员初始化并维护。*

* **功能路径**: `/api/v1/admin/users`
* **操作步骤**:
  1. **批量账号开通 (Batch Create Users)**: 整理全校师生的学号/工号以及姓名，通过系统提供的 `POST /users/batch` 接口，快速下发系统账号及初始密码。
  2. **信息维护**: 修改密码或对师生基本信息进行维护。

### 2.2 班级组织架构建设 (Class Management)
*组建班级是平台教学活动下发和学情监控的基础单位。*

* **功能路径**: `/api/v1/admin/classes`
* **操作步骤**:
  1. **组建新班级**: 指定年级、班级名称等信息。
  2. **分配任课教师**: 在创建或编辑时，将注册好的教师账号绑定到该班级。

### 2.3 教学大屏监控 (School Analytics & Marketplace)
*校管拥有和“教师”角色同等的“读取”和部分“业务”操作权限。*

* **支持的操作**:
  1. **洞察学情**: 通过全校数据面板监控学生掌握度（Mastery），并可查看各班级甚至具体学生活动的诊断分析图。
  2. **干预管理**: 查看并处理系统中产生的问题件与交互记录。
  3. **获取教学资源**: 从插件市场安装合适的辅助工具，支持教师进行教学创新。

## 3. 注意事项
* **范围限定**: 学校管理员所做的所有修改（班级、账号、插件等）仅对“本学校”生效。
* **协同办公**: 为了提升效率，对于部分教学教务职能较深的教师，也可以将其提升为校管。
